# Neovim設定 要件定義書
2025/12/7 snomu

## 1. 概要

### 1.1. 目的
Windows でよく利用する複数の機能をまとめたツールを作成する
今まではアプリケーションを切り替えながら各ツールを使っており、切替の手間があった
Neovim に複数の機能を持たせることにより、ツール間の切替の手間がなく、
シームレスに機能を切り替えられることで、効率を上げることを目的とする
- Windowsのエクスプローラー
- メモ帳
- ソースファイルエディタ
- ターミナル機能

### 1.2. 成果物
本プロジェクトで作成される**成果物**は以下の通りです。
- Neovim設定ファイル一式
- 要件定義書、設計書
- 初期セットアップ手順書

## 2. 機能要件

### 2.1. 基本機能
Neovimとして**必須**となる基本的な機能要件を記述します。

| ID | 機能名 | 概要 | 優先度 (高/中/低) |
| :--- | :--- | :--- | :--- |
| FR-001 | ファイル操作 | Windows のエクスプローラーのように快適にファイル操作できる | 高 |
| FR-002 | メモ機能 | メモをすぐに起動できて、書いて、参照して、閉じられる | 高 |
| FR-003 | ソース編集 | ソースコードがハイライトされ、開発・デバッグできるまではいかないものの、一定程度快適にソースコードを編集できる | 高 |
| FR-004 | ターミナル | ターミナルを呼び出し、簡単な操作が行える| 高 |
| FR-005 | クイックアクセス | 最近利用したフォルダ、よく利用する作業ディレクトリに高速にアクセスできる| 高 |
| FR-006 | ファイル名検索 | 現在ディレクトリ配下のファイル名検索が行える| 高 |
| FR-007 | grep検索 | 現在ディレクトリ配下のgrep検索が行える| 高 |
| FR-008 | エクスプローラー代替 | Windows の右クリックメニューで通常行うような各操作については、代替の手段を用意する | 高 |

## 3. 非機能要件

### 3.1. 性能要件
設定の**パフォーマンス**に関する要件を記述します。

| ID | 要件 | 目標値 | 測定方法 |
| :--- | :--- | :--- | :--- |
| NFR-001 | 起動速度 | 快適な速度で使える。 | `nvim --startuptime` コマンド |
| NFR-002 | プラグイン読み込み | 必要なプラグインのみ遅延ロードされること。 | `lazy.nvim` のプロファイル機能 |

### 3.2. 保守性要件
設定の**メンテナンス**に関する要件を記述します。

| ID | 要件 | 詳細 |
| :--- | :--- | :--- |
| NFR-010 | 言語 | 設定ファイルは**Lua**で記述すること。 |
| NFR-011 | モジュール化 | 設定は機能ごとにファイル分割し、`require()`で読み込むこと。 |
| NFR-012 | バージョン管理 | 設定ファイル一式は**Git**で管理し、GitHub等のリモートリポジトリにプッシュすること。 |

### 3.3. 環境要件
設定が動作する**環境**に関する要件を記述します。
Windows 11
Windows Terminal
Git Bash

## 4. 制約条件

### 4.1. 採用技術
ターミナルには Git Bash を利用
Neovim のプラグイン、および Nerd Fonts などのOS側に必要なツールのインストールが必要
